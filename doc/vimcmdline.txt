1. Overview						*vimcmdline_overview*

vimcmdline sends lines from either Vim or Neovim to a command line interpreter
(REPL application). There is support for Clojure, Golang, Haskell, JavaScript,
Julia, Jupyter, Lisp, Macaulay2, Matlab, Prolog, Python, Ruby, Sage, Scala,
Shell script, and Swift.

The interpreter may run in a Neovim built-in terminal (a Neovim buffer), an
external terminal emulator, or in a tmux pane. When running in either a Neovim
built-in terminal or an external terminal, the plugin runs one instance of the
REPL application for each file type. When running in a tmux pane, it runs one
REPL application per Vim instance.

==============================================================================
2. Usage						*vimcmdline_usage*

If you are editing one of the supported file types, in Normal mode press
|<LocalLeader>s| to start the interpreter, then use the mappings described in
|vimcmdline_mappings| to send code to it.

==============================================================================
3. Mappings						*vimcmdline_mappings*

The default mappings are listed below. Each can be changed by setting the
corresponding option (see |vimcmdline_options|).

Normal mode: >
	<LocalLeader>s		Start the interpreter
	<Space>			Send the current line and move to the next
	<LocalLeader><Space>	Send the current line and keep the cursor
	<LocalLeader>q		Send the quit command to the interpreter
<
For languages that can source chunks of code:

Visual mode: >
	<Space>			Send the selected text
<
Normal mode: >
	<LocalLeader>f		Send the entire file
	<LocalLeader>p		Send from the line to the end of paragraph
	<LocalLeader>b		Send block between the two closest marks
<
Code-block (cell) mappings, see |vimcmdline_blocks|: >
	<LocalLeader>c		Execute the current code block
	<LocalLeader>n		Execute the current code block and jump to next
	<LocalLeader>e		Execute from the current line to the block end
	<LocalLeader>]		Jump to the next block
	<LocalLeader>[		Jump to the previous block
<
==============================================================================
4. Code blocks						*vimcmdline_blocks*

A file can be divided into code blocks (also called "cells") delimited by a
separator line. The separator defaults to "# %%", matching the convention used
by Jupyter and VSCode, and is configurable with |cmdline_block_sep|.

The block mappings (|vimcmdline_mappings|) are defined only for file types that
provide a function to source a chunk of code (for example Python, Julia,
Shell). The separator is matched literally, so special characters in it are not
treated as a regular expression.

When the cursor is on a separator line, the current block is the region that
begins just after that separator.

5. Options						*vimcmdline_options*

Set these variables in your |vimrc| before the plugin is loaded.

Mapping options (value is the key sequence to use): >
	cmdline_map_start
	cmdline_map_send
	cmdline_map_send_and_stay
	cmdline_map_source_fun
	cmdline_map_send_paragraph
	cmdline_map_send_block
	cmdline_map_quit
	cmdline_map_exec_block
	cmdline_map_exec_block_and_jump
	cmdline_map_exec_to_end
	cmdline_map_next_block
	cmdline_map_prev_block
<
							*cmdline_app*
cmdline_app		Dictionary mapping a file type to the interpreter
			command to run for it. Example: >
				let cmdline_app = {}
				let cmdline_app['python'] = 'ptipython3'
				let cmdline_app['ruby']   = 'pry'
				let cmdline_app['sh']     = 'bash'
<
							*cmdline_in_buffer*
cmdline_in_buffer	Run the interpreter in a Neovim terminal buffer.
			Default: 1 on Neovim, 0 otherwise.

							*cmdline_vsplit*
cmdline_vsplit		Split the window vertically. Default: 0.

							*cmdline_split_topleft*
cmdline_split_topleft	When splitting vertically, place the interpreter
			window on the top left. Default: 0.

							*cmdline_esc_term*
cmdline_esc_term	Remap <Esc> to :stopinsert in Neovim's terminal.
			Default: 1.

							*cmdline_term_width*
cmdline_term_width	Initial width of the interpreter window or pane.
			Default: 40.

							*cmdline_term_height*
cmdline_term_height	Initial height of the interpreter window or pane.
			Default: 15.

							*cmdline_tmp_dir*
cmdline_tmp_dir		Temporary directory used to save files that are
			sourced by the interpreter. Default: a per-user,
			per-session directory under /tmp.

							*cmdline_outhl*
cmdline_outhl		Syntax highlight the interpreter output (Neovim).
			Default: 1.

							*cmdline_auto_scroll*
cmdline_auto_scroll	Keep the cursor at the end of the terminal (Neovim).
			Default: 1.

							*cmdline_block_sep*
cmdline_block_sep	Separator line delimiting code blocks. See
			|vimcmdline_blocks|. Default: '# %%'.

							*cmdline_external_term_cmd*
cmdline_external_term_cmd  Command used to run the interpreter in an external
			terminal emulator. "%s" is replaced with the command
			that starts the REPL in a tmux session. Examples: >
				let cmdline_external_term_cmd = "gnome-terminal -e '%s'"
				let cmdline_external_term_cmd = "xterm -e '%s' &"
<
==============================================================================
6. Colors						*vimcmdline_colors*

On Neovim, the interpreter output can be colorized. The colors are set with a
family of cmdline_color_* options, each of which accepts the hex code of a
foreground color, the ANSI number of a foreground color, or a complete
highlighting specification. To follow the current |colorscheme| instead, set: >
	let cmdline_follow_colorscheme = 1
<
See the README for the full list of cmdline_color_* options and examples.

==============================================================================
7. Adding support for a new language			*vimcmdline_newlang*

  1. Copy the script in the ftplugin directory whose language is closest to
     the one you want to support, and save it as "filetype_cmdline.vim" where
     "filetype" is the output of `echo &filetype` for that language.
  2. Edit the new script and change the values of its variables.
  3. Copy the closest script in the syntax directory and save it as
     "cmdlineoutput_app.vim" where "app" is the interpreter name.
  4. Edit the patterns used to recognize the input line and errors.
